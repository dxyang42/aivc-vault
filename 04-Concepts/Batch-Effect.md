# Batch Effect (批次效应)

> [[概念地图]] | [[scRNA-seq]] | [[数据预处理]] | [[术语统一规范]]

## 定义

**批次效应 (Batch Effect)** 是指由于非生物学因素导致的样本间系统性差异。这些差异来源于实验处理的技术变异，而非真实的生物学信号，会掩盖或扭曲真实的生物学发现。

### 核心概念

```
观测到的数据 = 真实生物学信号 + 批次效应 + 随机噪声

批次效应来源:
├── 技术批次
│   ├── 不同测序平台
│   ├── 不同实验日期
│   ├── 不同操作者
│   └── 不同试剂批次
│
└── 生物学批次
    ├── 不同供体
    ├── 不同组织来源
    └── 不同处理条件
```

### 为什么批次效应是问题

批次效应是单细胞数据分析中最常见的技术混淆因素。如果不加以处理，会导致：
- 假阳性发现（技术差异被误认为生物学差异）
- 降低研究的可重复性
- 掩盖真实的生物学信号

---

## 批次效应的来源

### 技术批次

| 来源 | 描述 | 影响程度 |
|------|------|----------|
| **测序平台** | Illumina NovaSeq vs HiSeq | 高 |
| **建库方案** | 10x Genomics vs Smart-seq2 | 高 |
| **实验日期** | 不同时间进行的实验 | 中-高 |
| **操作者** | 不同实验人员的技术差异 | 中 |
| **试剂批次** | 不同批次的酶和试剂 | 中 |
| **细胞捕获** | 细胞通量和捕获效率差异 | 中 |

### 生物学批次

| 来源 | 描述 | 处理策略 |
|------|------|----------|
| **供体差异** | 不同个体的遗传背景 | 保留（生物学信号） |
| **组织来源** | 不同器官或组织部位 | 保留（生物学信号） |
| **处理条件** | 样本采集和保存方式 | 校正（技术变异） |

### 区分技术批次与生物学变异

```
技术批次特征:
- 影响所有基因或大量基因
- 呈现系统性模式
- 与实验条件相关
- 需要校正

生物学变异特征:
- 与特定生物学过程相关
- 有功能意义
- 需要保留
```

## 批次效应的影响

### 1. 假阳性发现

```
场景: 比较疾病组 vs 对照组

问题设置:
- 疾病组样本: 2024年1月测序
- 对照组样本: 2024年6月测序

批次效应后果:
批次差异 (1月 vs 6月)
    ↓
被误认为疾病差异
    ↓
大量假阳性差异表达基因
    ↓
错误的生物学结论
```

### 2. 降低可重复性

| 问题 | 描述 |
|------|------|
| **研究间不一致** | 不同实验室的结果难以比较 |
| **元分析困难** | 合并多个数据集时产生偏差 |
| **验证失败** | 独立验证实验无法复现发现 |

### 3. 掩盖真实信号

```
真实生物学信号: 细胞类型A vs 细胞类型B的差异
批次效应信号: 批次1 vs 批次2的差异

当批次效应 >> 生物学信号时:
- 细胞类型在批次间混杂
- 聚类结果按批次而非类型分组
- 真实的细胞类型差异被淹没
```

## 批次校正方法

### 线性方法

#### ComBat (Combat Seq)

**原理**: 使用经验贝叶斯框架调整批次效应。

```
模型: Y_ijg = α_g + X_ijβ_g + γ_ig + δ_igε_ijg

其中:
- Y_ijg: 批次i中样本j的基因g的表达
- α_g: 基因g的总体均值
- X_ij: 生物学协变量（需要保留的）
- β_g: 生物学效应
- γ_ig, δ_ig: 批次i的加性和乘性效应
- ε_ijg: 随机误差

ComBat估计批次参数(γ, δ)，然后校正:
Y_ijg_corrected = (Y_ijg - γ̂_ig) / δ̂_ig × σ̂_g + μ̂_g
```

**特点**:
- 成熟稳定，广泛使用
- 假设批次效应是线性的
- 需要指定生物学协变量

**适用场景**:
- 已知批次标签
- 批次间有共享的生物学群体
- 线性批次效应占主导

### 非线性方法

#### Harmony

**原理**: 在嵌入空间中使用软聚类进行迭代校正。

```
算法流程:
1. PCA降维得到细胞嵌入
2. 迭代优化:
   a. 软聚类: 为每个细胞分配聚类概率
   b. 计算每个批次在每个聚类的质心
   c. 校正: 将批次质心对齐到全局质心
3. 直到收敛

关键创新:
- 软聚类允许细胞属于多个聚类
- 仅校正批次间差异，保留生物学变异
```

**特点**:
- 处理非线性批次效应
- 保留细胞亚型结构
- 计算效率高

**适用场景**:
- 复杂批次结构
- 细胞类型组成差异大
- 需要保留细微细胞亚型

#### Scanorama

**原理**: 基于互最近邻(MNN)的数据集整合。

```
算法流程:
1. 对每个数据集分别进行PCA
2. 识别互最近邻对(MNN):
   - 细胞A在数据集1中
   - 细胞B在数据集2中
   - A是B在数据集1中的最近邻
   - B是A在数据集2中的最近邻
3. 计算批次校正向量:
   基于MNN对的表达差异
4. 应用校正:
   使用局部加权校正每个细胞

假设: MNN对代表相同的生物学状态
```

**特点**:
- 不需要预先对齐的细胞类型
- 处理异质数据集
- 保留生物学变异

**适用场景**:
- 数据集间细胞类型组成不同
- 部分重叠的细胞群体
- 异质性高的数据

### 深度学习方法

#### scVI (Single-cell Variational Inference)

**原理**: 使用变分自编码器同时建模生物学变异和批次效应。

```
模型结构:
编码器: q(z|x, b)  # 给定表达x和批次b，推断潜在状态z
解码器: p(x|z, b)   # 从潜在状态z和批次b重构表达

批次处理:
- 批次信息b输入编码器和解码器
- 模型学习分离生物学变异(z)和批次效应(b)
- 批次校正的表达: 使用统一批次解码

损失函数:
ELBO = E_q(z|x,b)[log p(x|z,b)] - KL(q(z|x,b) || p(z))
```

**特点**:
- 端到端学习
- 处理dropout和过离散
- 可生成批次校正的表达矩阵

**适用场景**:
- 大规模数据集
- 复杂的批次结构
- 需要不确定性估计

## 评估指标

### kBET (k-Nearest Neighbor Batch Effect Test)

**原理**: 检验批次标签在k近邻中的分布是否随机。

```
计算方法:
1. 对每个细胞，找到其k个最近邻
2. 统计这k个邻居中各批次的细胞数
3. 与随机期望比较（卡方检验）
4. 计算拒绝率（批次混合差的细胞比例）

解释:
- kBET ≈ 1: 批次混合差，批次效应严重
- kBET ≈ 0: 批次混合好，校正成功
```

### LISI (Local Inverse Simpson's Index)

**原理**: 测量局部邻域的多样性。

```
计算方法:
1. 对每个细胞，基于距离定义局部邻域
2. 计算逆辛普森指数:
   LISI = 1 / Σ(p_i)²
   其中p_i是批次i在邻域中的比例

解释:
- LISI_batch ≈ 1: 邻域内批次单一（批次效应）
- LISI_batch ≈ N_batches: 邻域内批次均匀混合（校正成功）
- LISI_cell_type: 同样计算细胞类型多样性

理想情况: LISI_batch高，LISI_cell_type低
（批次混合好，细胞类型分离）
```

### ARI (Adjusted Rand Index)

**原理**: 比较聚类结果与真实标签的一致性。

```
应用场景:
- 比较校正前后的聚类结果
- 评估批次校正是否保留了生物学结构

计算:
ARI = (RI - Expected_RI) / (Max_RI - Expected_RI)

其中RI是Rand Index，衡量两个聚类的一致性

解释:
- ARI = 1: 完美一致
- ARI = 0: 随机一致
- ARI < 0: 比随机还差
```

### 可视化评估

```
UMAP可视化检查:

校正前:
颜色=批次 → 明显分离的簇
颜色=细胞类型 → 混杂在批次簇中

校正后:
颜色=批次 → 均匀混合
颜色=细胞类型 → 清晰分离的簇
```

## 方法选择指南

| 场景 | 推荐方法 | 理由 |
|------|----------|------|
| 简单线性批次 | ComBat | 稳定、可解释 |
| 复杂非线性批次 | Harmony | 保留亚型结构 |
| 异质数据集 | Scanorama | 处理部分重叠 |
| 大规模数据 | scVI | 端到端、可扩展 |
| 未知批次标签 | scVI/Harmony | 不需要批次标签 |

### 最佳实践

```
1. 实验设计阶段:
   - 尽量平衡批次和生物学条件
   - 每个批次包含所有关键细胞类型
   - 设置批次内重复

2. 质控阶段:
   - 可视化检查批次分布
   - 计算批次效应指标

3. 校正阶段:
   - 尝试多种方法比较
   - 验证校正后生物学信号保留
   - 避免过度校正

4. 验证阶段:
   - 使用独立数据集验证
   - 检查已知生物学标记
   - 评估下游任务性能
```
