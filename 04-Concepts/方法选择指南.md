# 方法选择指南

> 如何为不同的Perturbation预测任务选择合适的方法

## 快速决策流程

```
开始
  │
  ├─→ 数据类型?
  │     ├─→ 仅RNA → 继续
  │     ├─→ RNA+Protein → totalVI
  │     └─→ 多组学 → MultiVI
  │
  ├─→ 扰动类型?
  │     ├─→ 单基因 → GEARS / scGen
  │     ├─→ 组合基因 → CPA / GEARS
  │     ├─→ 化合物 → CPA
  │     └─→ 未知类型 → AlphaCell / CellHermes
  │
  ├─→ 是否有参考数据?
  │     ├─→ 是 → scArches (迁移学习)
  │     └─→ 否 → 继续
  │
  ├─→ 数据规模?
  │     ├─→ 大规模 (>100万细胞) → STATE / scVI
  │     └─→ 小规模 → 继续
  │
  └─→ 计算资源?
        ├─→ GPU可用 → GEARS / CPA / AlphaCell
        └─→ CPU only → scGen / Cell Oracle
```

## 按任务类型选择

### 1. 单基因扰动预测

| 优先级 | 方法 | 理由 |
|--------|------|------|
| ⭐⭐⭐ | **GEARS** | GNN+知识图谱，组合预测能力强 |
| ⭐⭐⭐ | **scGen** | 经典VAE方法，稳定可靠 |
| ⭐⭐ | **scVI** | 基础方法，生态完善 |
| ⭐⭐ | **CPA** | 支持剂量-响应 |

### 2. 组合扰动预测

| 优先级 | 方法 | 理由 |
|--------|------|------|
| ⭐⭐⭐ | **CPA** | 专为组合扰动设计 |
| ⭐⭐⭐ | **GEARS** | 预测非加性相互作用 |
| ⭐⭐ | **AlphaCell** | 零样本组合预测 |

### 3. 化合物/药物扰动

| 优先级 | 方法 | 理由 |
|--------|------|------|
| ⭐⭐⭐ | **CPA** | 剂量-响应建模 |
| ⭐⭐ | **scGen** | 跨条件预测 |
| ⭐⭐ | **CellFlow** | 连续剂量空间 |

### 4. 空间转录组扰动

| 优先级 | 方法 | 理由 |
|--------|------|------|
| ⭐⭐ | **MultiVI** | 支持空间数据整合 |
| ⭐ | **scVI** + 空间工具 | 需要额外处理 |

### 5. 零样本预测 (未见扰动)

| 优先级 | 方法 | 理由 |
|--------|------|------|
| ⭐⭐⭐ | **AlphaCell** | 世界模型，零样本能力强 |
| ⭐⭐⭐ | **CellHermes** | 语言模型，文本描述扰动 |
| ⭐⭐ | **scArches** | 迁移学习到参考图谱 |

## 按数据规模选择

### 小规模数据 (<10,000 细胞)

| 方法 | 适用场景 |
|------|----------|
| **scGen** | 简单扰动预测 |
| **Cell Oracle** | GRN分析，可解释性强 |
| **scVI** | 批次校正+降维 |

**建议**: 数据有限时选择简单模型，避免过拟合

### 中等规模数据 (10,000 - 100,000 细胞)

| 方法 | 适用场景 |
|------|----------|
| **GEARS** | 单/组合基因扰动 |
| **CPA** | 组合扰动+剂量 |
| **scANVI** | 需要细胞注释 |

### 大规模数据 (>100,000 细胞)

| 方法 | 适用场景 |
|------|----------|
| **STATE** | 超大规模，Transformer架构 |
| **scVI** | GPU加速，可扩展 |
| **AlphaCell** | 高效世界模型 |

**建议**: 使用GPU加速，考虑数据采样策略

## 按计算资源选择

### GPU可用 (推荐)

| 方法 | GPU内存需求 | 训练时间 |
|------|-------------|----------|
| **STATE** | 16GB+ | 数小时-数天 |
| **AlphaCell** | 8GB+ | 数小时 |
| **GEARS** | 4GB+ | 1-2小时 |
| **CPA** | 4GB+ | 1-2小时 |
| **CellHermes** | 16GB+ | 数小时-数天 |

### CPU Only

| 方法 | 适用场景 |
|------|----------|
| **scGen** | 轻量级VAE |
| **Cell Oracle** | GRN方法 |
| **scVI** (小数据) | 基础分析 |

### TPU/多节点

| 方法 | 扩展性 |
|------|--------|
| **STATE** | 原生支持大规模训练 |
| **scVI** | 可分布式训练 |

## 按数据类型选择

### 仅scRNA-seq

```
推荐顺序:
1. GEARS (单/组合基因)
2. CPA (组合+剂量)
3. scGen (简单场景)
4. scVI (基础分析)
```

### CITE-seq (RNA + Protein)

```
推荐顺序:
1. totalVI (端到端多模态)
2. MultiVI (如果有ATAC)
3. scVI + 单独Protein分析
```

### 多组学 (RNA + ATAC + Protein)

```
推荐顺序:
1. MultiVI (马赛克整合)
2. 分别分析后整合
```

## 特殊场景

### 有参考图谱

| 场景 | 方法 | 说明 |
|------|------|------|
| 映射到参考 | **scArches** | 快速映射，保留参考知识 |
| 注释转移 | **scANVI** | 半监督注释 |

### 需要可解释性

| 方法 | 可解释性来源 |
|------|-------------|
| **Cell Oracle** | GRN网络 |
| **GEARS** | 知识图谱 |
| **CPA** | 解耦潜在空间 |

### 需要不确定性估计

| 方法 | 不确定性来源 |
|------|-------------|
| **scVI** | 变分推断后验 |
| **totalVI** | 概率生成模型 |
| **CellFlow** | 流匹配似然 |

## 组合推荐

### 完整分析流程

```
1. 数据预处理
   └─→ scVI (批次校正)

2. 细胞注释
   └─→ scANVI (半监督注释)

3. 扰动预测
   ├─→ GEARS (基因扰动)
   └─→ CPA (组合扰动)

4. 结果验证
   └─→ scArches (映射到参考)
```

### 快速探索

```
1. scVI (快速降维可视化)
2. scGen (快速扰动预测)
```

### 深度分析

```
1. totalVI/MultiVI (多模态整合)
2. AlphaCell/CellHermes (零样本预测)
3. Cell Oracle (机制解析)
```

## 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 小数据用大模型 | 过拟合 | 选择简单模型 |
| 忽略批次效应 | 技术噪声 | 先用scVI校正 |
| 多模态分开分析 | 丢失关联 | 使用totalVI/MultiVI |
| 零样本用监督方法 | 无法泛化 | 选择AlphaCell/CellHermes |
