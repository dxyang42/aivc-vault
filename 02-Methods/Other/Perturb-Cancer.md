## 核心技术

### 架构设计

Perturb-Cancer 是一个**患者来源癌症模型的扰动预测框架**，整合患者异质性和药物响应预测：

```

│                    Perturb-Cancer 架构                               │
├─────────────────────────────────────────────────────────────────────┤

│   │   │ • 患者来源细胞系                                     │  │  │
```

## 训练数据

| 数据属性 | 规模 |
|---------|------|
| **患者来源类器官** | 5,000+ |
| **药物筛选数据** | 500+ 化合物 |
| **癌症类型** | 25+ 种 |
| **患者样本** | 2,000+ |
| **单细胞数据** | 100万+ 细胞 |

## 扰动预测能力

### 支持的扰动类型

| 扰动类型 | 支持情况 | 说明 |
|---------|---------|------|
| **化疗药物** | 完全支持 | 铂类、紫杉醇等 |
| **靶向治疗** | 完全支持 | EGFR, ALK, BRAF抑制剂等 |
| **免疫治疗** | 支持 | 检查点抑制剂响应预测 |
| **联合用药** | 核心优势 | 多药物组合优化 |
| **基因编辑** | 支持 | CRISPR筛选数据整合 |

### 预测能力矩阵

| 预测任务 | 准确性 | 临床应用 |
|---------|--------|---------|
| **药物敏感性分类** | AUC 0.87 | 指导用药选择 |
| **IC50估计** | R² 0.72 | 剂量优化 |
| **耐药机制预测** | 准确率 82% | 克服耐药策略 |
| **联合用药协同** | 准确率 78% | 组合疗法设计 |
| **患者分层** | NMI 0.85 | 临床试验设计 |

## 核心技术特点

### 1. 从PDMs到患者的迁移学习

```
┌─────────────────────────────────────────────────────────────┐
│              PDMs到患者的迁移学习                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   训练阶段 (PDMs):                                          │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                     │  │
│   │   患者来源类器官 ──→ 大规模药物筛选 ──→ 响应数据   │  │
│   │                                                     │  │
│   │   优势: 可高通量测试数百种药物                      │  │
│   │   局限: 不能完全代表患者肿瘤                        │  │
│   │                                                     │  │
│   └─────────────────────────────────────────────────────┘  │
│                              ↓                              │
│   迁移到临床患者:                                           │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                     │  │
│   │   原发肿瘤分子特征 ──→ UCE/基因嵌入 ──→ 患者表示   │  │
│   │                                                     │  │
│   │   应用预训练模型预测药物响应                        │  │
│   │                                                     │  │
│   │   验证: 与临床响应记录对比                          │  │
│   │                                                     │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 多组学数据整合

| 数据类型 | 整合方式 | 预测贡献 |
|---------|---------|---------|
| **基因组** | 突变编码器 | 驱动基因、耐药突变 |
| **转录组** | 表达编码器 | 通路活性、细胞状态 |
| **表观遗传** | 甲基化/ATAC编码器 | 基因调控状态 |
| **蛋白质组** | 蛋白活性推断 | 信号通路活性 |

### 3. 动态响应建模

- **时序预测**: 预测药物响应的时间动态
- **适应性进化**: 预测耐药克隆的出现
- **微环境交互**: 建模肿瘤-微环境相互作用

## 应用场景

### 1. 精准肿瘤学

```
临床工作流程:
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   患者肿瘤样本 ──→ 分子 profiling ──→ Perturb-Cancer预测   │
│                                          ↓                  │
│   治疗方案 ←── 敏感性排序 ←── 多药物响应预测               │
│                                          ↓                  │
│   治疗响应监测 ←── 耐药预警 ←── 动态模型更新               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 药物重定位

- 预测已批准药物对新癌症类型的疗效
- 识别罕见癌症的治疗选择
- 加速孤儿药开发

### 3. 联合疗法设计

| 应用 | 描述 |
|------|------|
| **协同预测** | 识别协同作用的药物对 |
| **序贯治疗** | 优化给药顺序 |
| **耐药预防** | 设计防止耐药的组合 |

### 4. 临床试验优化

- 患者分层：识别最可能响应的患者亚群
- 生物标志物发现：预测响应的分子标志物
- 适应性试验设计：动态调整试验方案

## 优势与局限

### 优势
- 专门针对患者异质性设计
- 整合多组学数据
- 直接面向临床应用
- 支持联合用药优化
- 持续学习临床反馈

### 局限
- 依赖高质量PDMs数据
- 需要患者分子 profiling
- 模型解释性有待提升
- 罕见癌症数据有限
- 长期耐药预测挑战
