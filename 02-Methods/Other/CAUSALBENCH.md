│   │                                                                 │  │

### 架构设计

CausalBench提供**全面的基准测试框架**：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    CausalBench 基准套件架构                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   组件1: 数据集集合                                                      │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                 │  │
│   │  两个大规模真实世界数据集:                                       │  │
│   │                                                                 │  │
│   │  数据集1: 遗传扰动数据                                           │  │
│   │  ├── 细胞系: K562, RPE1                                         │  │
│   │  ├── 扰动: 全基因组CRISPR筛选                                    │  │
│   │  ├── 读数: 单细胞转录组                                          │  │
│   │  └── 规模: 数百万细胞                                            │  │
│   │                                                                 │  │
│   │  数据集2: 药物扰动数据                                           │  │
│   │  ├── 细胞系: 多种癌细胞系                                        │  │
│   │  ├── 扰动: 多种药物处理                                          │  │
│   │  ├── 读数: 单细胞转录组                                          │  │
│   │  └── 规模: 大规模                                                │  │
│   │                                                                 │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                              ↓                                          │
│   组件2: 生物启发评估指标                                                │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                 │  │
│   │  区别于传统指标 (AUPR, AUROC):                                   │  │
│   │                                                                 │  │
│   │  1. 通路富集指标                                                 │  │
│   │     - 推断的边是否在已知通路中富集                               │  │
│   │     - 与GO、KEGG等数据库比较                                     │  │
│   │                                                                 │  │
│   │  2. 中心性保留                                                   │  │
│   │     - 关键调控因子是否被正确识别                                 │  │
│   │     - 与已知的master regulators比较                              │  │
│   │                                                                 │  │
│   │  3. 干预分布度量                                                 │  │
│   │     - 基于分布的干预效果评估                                     │  │
│   │     - 超越简单的边存在性判断                                     │  │
│   │                                                                 │  │
│   │  4. 鲁棒性指标                                                   │  │
│   │     - 跨细胞类型的稳定性                                         │  │
│   │     - 跨批次的稳定性                                             │  │
│   │                                                                 │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                              ↓                                          │
│   组件3: 标准化评估流程                                                  │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                 │  │
│   │  提供统一的:                                                    │  │
│   │  ├── 数据预处理流程                                              │  │
│   │  ├── 训练/测试分割策略                                           │  │
│   │  ├── 评估脚本和工具                                              │  │
│   │  └── 结果报告格式                                                │  │
│   │                                                                 │  │
│   │  支持的方法类型:                                                │  │
│   │  ├── 基于约束的方法 (PC, GES)                                    │  │
│   │  ├── 基于分数的方法 (NOTEARS, DAG-GNN)                          │  │
│   │  ├── 基于干预的方法 (IGSP)                                       │  │
│   │  └── 深度学习方法 (变分推断、强化学习)                           │  │

│   └── 最佳实践指南                                                    │
│                                                                         │

```

## 关键特性

### 数据集统计

| 特征 | 规模 |
|------|------|
| **数据集数量** | 2个主要数据集 |
| **细胞数量** | 数百万 |
| **基因数量** | 20,000+ |
| **扰动数量** | 数千 |
| **细胞系** | 多种 |

### 评估指标

| 指标类型 | 描述 | 生物学意义 |
|---------|------|-----------|
| **通路富集** | 推断边在已知通路中的富集程度 | 与现有知识一致性 |
| **中心性保留** | 关键调控因子的识别准确性 | 功能重要性 |
| **干预效果** | 基于分布的干预预测能力 | 实际预测价值 |
| **鲁棒性** | 跨条件稳定性 | 泛化能力 |

## 技术优势

### 与现有基准对比

| 特性 | CausalBench | 合成数据基准 | 小尺度真实数据 |
|------|-------------|-------------|---------------|
| **数据规模** | 大规模 | 可变 | 小 |
| **真实性** | 真实世界 | 合成 | 真实 |
| **生物学相关性** | 高 | 低 | 中 |
| **评估指标** | 生物启发 | 统计指标 | 混合 |
| **可重复性** | 高 | 高 | 中 |

### 核心创新

1. **真实世界数据**: 使用大规模真实扰动数据而非合成数据

2. **生物启发指标**: 设计反映生物学意义的评估指标

3. **分布级评估**: 超越简单的边存在性判断，评估干预效果分布

4. **标准化流程**: 提供端到端的标准化评估流程

## 应用场景

### 对方法开发者的价值

1. **公平比较**: 在相同数据上比较不同方法
2. **诊断工具**: 识别方法的优势和弱点
3. **改进指导**: 基于指标反馈指导方法改进

### 对实践者的价值

1. **方法选择**: 根据需求选择合适的方法
2. **性能预期**: 了解在真实数据上的预期性能
3. **最佳实践**: 学习领域内的最佳实践

## 竞赛与挑战

### ICLR 2023 Challenge

CausalBench举办了ICLR 2023竞赛：
- **任务**: 从单细胞扰动数据推断基因网络
- **参与者**: 全球多个研究团队
- **成果**: 推动了方法发展和社区交流
