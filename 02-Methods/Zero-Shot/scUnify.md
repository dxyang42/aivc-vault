> Unified Zero-shot Inference Pipeline for Single-Cell Foundation Models

## 核心思想

scUnify提供了一个**统一的零样本推理管道**，支持多种单细胞基础模型的比较和集成。通过标准化接口和评估协议，该方法使研究者能够公平地比较不同模型在扰动预测任务上的零样本性能。

### 关键创新

1. **统一接口**: 为不同基础模型提供一致的API
2. **多模型支持**: 支持scGPT, Geneformer, scFoundation等
3. **标准化评估**: 统一的评估指标和数据预处理
4. **模型集成**: 组合多个模型的预测提高准确性

## 零样本推理流程

```
输入: 单细胞数据 + 扰动列表
    ↓
数据标准化
├── 基因名称统一 (HGNC标准)
├── 表达量归一化
└── 格式转换
    ↓
模型推理 (并行)
├── Model A: scGPT → 预测 A
├── Model B: Geneformer → 预测 B
├── Model C: scFoundation → 预测 C
└── ...
    ↓
预测集成 (可选)
├── 简单平均: (A + B + C) / 3
├── 加权平均: w1*A + w2*B + w3*C
└── 堆叠: Meta-learner(A, B, C)
    ↓
输出: 最终预测 + 不确定性估计
```

## 输入/输出

### 输入
| 类型 | 描述 |
|------|------|
| 表达数据 | 单细胞基因表达矩阵 (AnnData格式) |
| 扰动信息 | 要预测的扰动列表 |
| 模型列表 | 要使用的模型名称列表 |
| 配置参数 | 推理和集成策略配置 |

### 输出
| 类型 | 描述 |
|------|------|
| 预测矩阵 | 每个模型和集成的预测结果 |
| 性能报告 | 各模型性能对比 |
| 不确定性 | 模型间方差作为不确定性 |

## 优势与局限

### 优势
- 公平比较不同基础模型
- 降低模型选择成本
- 集成预测提高鲁棒性
- 标准化促进社区协作

### 局限
- 需要维护多个模型适配器
- 不同模型的计算资源需求差异大
- 集成策略选择需要经验
- 难以处理模型间的系统性偏差

## 应用场景

1. **模型选择**: 为特定任务选择最佳基础模型
2. **基准测试**: 公平评估新模型的性能
3. **集成预测**: 组合多模型优势提高准确性
4. **研究复现**: 标准化实验设置便于复现
