# 技术路线对比

## Perturbation Prediction 技术路线

### 概述

Perturbation Prediction（扰动效果预测）是单细胞分析的核心任务之一，旨在预测细胞在基因敲除、药物处理等扰动条件下的转录组响应。不同技术路线在这个任务上各有优劣，本节专门对比分析。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Perturbation Prediction 技术路线全景              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌────────────┐ │
│   │ VAE-based   │  │ GNN-based   │  │Flow-based   │  │Transformer │ │
│   │             │  │             │  │             │  │  -based    │ │
│   │ • scGen     │  │ • GEARS     │  │ • CellFlow  │  │ • STATE    │ │
│   │ • CPA       │  │ • CellOracle│  │ • scDFM     │  │ • X-Pert   │ │
│   │             │  │             │  │ • SCALE     │  │ • scGPT    │ │
│   │ 概率框架    │  │ 先验知识    │  │ 连续建模    │  │ 预训练表示 │ │

```

### 路线1: VAE-based Perturbation Prediction

#### 代表方法
scGen (2019), CPA (2023)

#### 核心思想
VAE-based方法将扰动预测建模为潜在空间中的向量运算。学习一个从对照细胞到扰动后细胞的潜在空间映射向量，新细胞的预测通过潜在空间向量加法完成。

```
对照细胞表达        潜在空间          扰动后细胞表达
     │                 │                  │
     │    ┌───────┐    │     ┌───────┐    │
     └───→│Encoder│────┼────→│Decoder│────┘
          └───┬───┘    │     └───────┘
              │        │
              ▼        │
           z_control   │
              +        │
           delta_z     │ (扰动向量)
              │        │
              ▼        │
           z_perturbed │
```

#### 在Perturbation Prediction上的优势

| 维度 | 表现 | 说明 |
|------|------|------|
| **不确定性量化** | 优秀 | 概率框架天然支持置信度估计 |
| **计算效率** | 高 | 单次前向传播，推理速度快 |
| **数据效率** | 中等 | 相比GNN需要更多数据，但比Flow-based少 |
| **组合扰动** | CPA支持 | 解耦表示支持组合预测 |
| **剂量响应** | CPA支持 | 连续剂量建模能力 |

#### 在Perturbation Prediction上的局限

| 局限 | 说明 |
|------|------|
| **生成质量** | 相比Flow-based方法，生成样本质量较低 |
| **高斯先验约束** | 潜在空间的分布假设可能限制表达能力 |
| **组合泛化** | scGen原生不支持组合扰动 |
| **连续动态** | 不支持细胞状态的连续插值 |

---

### 路线2: GNN-based Perturbation Prediction

#### 代表方法
GEARS (2022), Cell Oracle (2022), Celcomen (2025)

#### 核心思想
GNN-based方法显式利用基因关系网络（如PPI网络），通过图神经网络传播扰动信号。扰动基因作为图上的信号源，通过网络传播影响其他基因。

```
        扰动基因A
            │
            ▼
    ┌───────────────┐
    │   基因关系图   │
    │  ┌───┐ ┌───┐  │
    │  │ A │←│ B │  │
    │  └─┬─┘ └───┘  │
    │    │   ┌───┐  │
    │    └──→│ C │  │
    │        └─┬─┘  │
    │      ┌───┘    │
    │      ▼        │
    │    ┌───┐      │
    │    │ D │      │
    │    └───┘      │
    └───────────────┘
            │
            ▼
    预测表达变化
```

#### 在Perturbation Prediction上的优势

| 维度 | 表现 | 说明 |
|------|------|------|
| **机制可解释性** | 优秀 | 注意力权重反映基因重要性 |
| **组合效应建模** | GEARS优秀 | 显式建模非加性相互作用 |
| **先验知识整合** | 优秀 | 利用PPI等生物学网络 |
| **样本效率** | 高 | 网络结构减少数据需求 |
| **新基因预测** | 部分支持 | 可预测网络中未见基因的间接效应 |

#### 在Perturbation Prediction上的局限

| 局限 | 说明 |
|------|------|
| **网络依赖** | 预测质量受限于输入网络的质量和完整性 |
| **网络偏见** | 现有网络可能偏向研究较多的基因和通路 |
| **动态建模** | 不支持细胞状态的连续动态变化 |
| **空间信息** | 仅Celcomen支持空间转录组 |

### 路线3: Flow-based Perturbation Prediction

#### 代表方法
CellFlow (2025), scDFM (2026), SCALE (2026), AlphaCell (2026), X-Cell (2026)

#### 核心思想
Flow-based方法学习细胞状态空间中的速度场，将扰动预测建模为从对照状态到扰动状态的连续流动。基于最优传输理论，学习最可能的细胞状态转换路径。

```
细胞状态空间

对照状态        中间状态        扰动后状态
   ●────────────●────────────●
   │            │            │
   │    速度场   │            │
   │     ↓      │            │
   │     ↓      │            │
   │     ↓      │            │
   └────→┴────→┴────→┘

   流匹配: 学习速度场 v(x,t)
   扩散模型: 学习去噪过程
```

#### 在Perturbation Prediction上的优势

| 维度 | 表现 | 说明 |
|------|------|------|
| **生成质量** | 优秀 | 当前最高质量的单细胞生成 |
| **连续插值** | 优秀 | 支持观察中间过渡状态 |
| **不确定性量化** | 良好 | 通过多采样估计不确定性 |
| **可解释路径** | 优秀 | 转换路径具有生物学意义 |
| **大规模数据** | SCALE优秀 | 可扩展至亿级数据 |

#### 在Perturbation Prediction上的局限

| 局限 | 说明 |
|------|------|
| **计算成本** | 推理需要多步采样，计算开销大 |
| **训练数据需求** | 需要大量配对扰动-响应对 |
| **推理速度** | 相比单次前向传播方法慢10-100倍 |
| **训练稳定性** | 某些架构可能存在训练不稳定问题 |

### 路线4: Transformer-based Perturbation Prediction

#### 代表方法
STATE (2025), X-Pert (2025), scFoundation (2024), scGPT (2023)

#### 核心思想
Transformer-based方法通过大规模预训练学习基因的通用表示，将扰动预测转化为条件生成任务。利用自注意力机制捕捉基因间的长程依赖关系。

```
输入序列: [CLS] gene_A:0.5 gene_B:2.3 ... [PERT] KO_gene_X [MASK]...
                │
                ▼
    ┌───────────────────────┐
    │    Transformer Encoder │
    │  ┌─────────────────┐  │
    │  │  Self-Attention │  │
    │  │  ┌───┐ ┌───┐   │  │
    │  │  │ Q │ │ K │   │  │
    │  │  └─┬─┘ └─┬─┘   │  │
    │  │    └──→┌┴┐    │  │
    │  │        │V│    │  │
    │  │        └┬┘    │  │
    │  └─────────┼─────┘  │
    │            ▼        │
    │      FFN Layer      │
    └──────────┬──────────┘
               │
               ▼
    预测被掩码基因的表达值
```

#### 在Perturbation Prediction上的优势

| 维度 | 表现 | 说明 |
|------|------|------|
| **零样本预测** | STATE/X-Pert优秀 | 无需针对特定扰动训练 |
| **迁移能力** | 优秀 | 预训练模型可迁移到多种任务 |
| **上下文感知** | 优秀 | 自注意力捕捉长程依赖 |
| **推理速度** | 高 | 单次前向传播 |
| **大规模数据利用** | 优秀 | 可利用海量未标注数据 |

#### 在Perturbation Prediction上的局限

| 局限 | 说明 |
|------|------|
| **计算资源需求** | 预训练需要大量GPU资源 |
| **可解释性** | 深度Transformer内部机制难解释 |
| **预训练偏见** | 预训练数据的质量和偏见影响表现 |
| **不确定性量化** | 相比VAE和Flow方法，不确定性估计较弱 |

## 路线1: 流匹配/扩散模型

### 代表方法
CellFlow, DC-DSB, scDFM, SCALE, CFM-GP, scPPDM, AlphaCell, X-Cell

### 核心思想

流匹配和扩散模型将细胞状态转换建模为一个连续的生成过程。想象细胞状态空间是一个高维空间，扰动作用相当于在这个空间中推动细胞从一个位置移动到另一个位置。这些方法学习一个速度场或噪声调度，描述细胞状态如何随时间演化。

具体来说，流匹配方法学习一个速度场网络，它告诉我们在任意时刻、任意细胞状态下，细胞应该向哪个方向、以多大速度移动。扩散模型则通过逐步添加和去除噪声的过程，学习从对照状态生成扰动后状态。

### 优势

- **生成质量高**: 能够生成高质量的细胞状态，捕捉数据的复杂分布
- **连续插值**: 支持细胞状态之间的连续插值，可观察中间过渡状态
- **概率建模**: 天然支持不确定性量化，提供预测的置信区间
- **可解释路径**: 生成的中间路径具有生物学意义，可解释性强
- **理论基础扎实**: 基于最优传输理论和随机微分方程，数学性质良好

### 局限

- **计算成本高**: 推理需要多步采样或ODE求解，计算开销较大
- **训练数据需求大**: 需要大量配对的扰动-响应对进行训练
- **训练稳定性**: 某些架构可能存在训练不稳定的问题
- **推理速度慢**: 相比单次前向传播的方法，推理速度较慢

### 适用场景

- 需要高质量生成样本的虚拟细胞实验
- 需要观察细胞状态连续转换过程的研究
- 对预测不确定性有要求的应用场景
- 有足够计算资源和训练数据的场景

## 路线2: Transformer 基础模型

### 代表方法
scGPT, scBERT, GenePT, scFoundation, UCE, STATE, X-Pert

### 核心思想

Transformer基础模型借鉴自然语言处理中的大规模预训练范式，将单细胞数据视为基因词汇的序列。通过在海量单细胞数据上进行自监督预训练，学习基因的通用表示和基因间的复杂交互关系。

这些模型通常采用掩码预测任务（随机掩码部分基因，预测其表达值）进行预训练。预训练完成后，模型可以作为特征提取器用于下游任务，或通过微调适应特定应用。

### 优势

- **迁移能力强**: 预训练模型可迁移到多种下游任务
- **大规模数据利用**: 能够利用海量未标注数据进行预训练
- **上下文感知**: 自注意力机制捕捉基因间的长程依赖关系
- **零样本能力**: 部分方法（如GenePT）无需训练即可使用
- **持续改进**: 随着数据量增加，模型性能可持续提升

### 局限

- **计算资源需求高**: 大规模预训练需要大量GPU资源
- **可解释性有限**: 深度Transformer的内部机制较难解释
- **数据偏见**: 预训练数据的质量和偏见会影响模型表现
- **领域适配**: 特定领域可能需要额外的微调

### 适用场景

- 需要通用细胞表示的大规模细胞图谱项目
- 多种下游任务的统一特征提取
- 跨物种、跨组织的细胞类型比较
- 有充足计算资源进行预训练或微调的场景

## 路线3: 变分自编码器(VAE)

### 代表方法
scVI, scGen, CPA, scANVI, totalVI, MultiVI

### 核心思想

VAE基于概率生成模型框架，假设观测数据（基因表达）由潜在的低维细胞状态通过随机过程生成。模型包含编码器（将观测映射到潜在空间）和解码器（从潜在空间重建观测）。

通过变分推断，VAE学习潜在空间的概率分布，支持不确定性量化。scVI系列方法特别针对单细胞数据的稀疏性和过离散性，采用零膨胀负二项分布建模基因表达。

### 优势

- **概率框架**: 完整的不确定性量化能力
- **理论基础扎实**: 基于贝叶斯统计，数学性质良好
- **批次校正**: 内置批次效应校正能力
- **可扩展性**: scvi-tools生态系统提供完善的工具支持
- **端到端**: 无需启发式预处理

### 局限

- **生成质量**: 相比流模型和扩散模型，生成质量可能较低
- **潜在空间约束**: 高斯先验假设可能限制表达能力
- **计算复杂度**: 变分推断的计算成本较高
- **架构相对固定**: 相比Transformer，架构创新空间有限

### 适用场景

- 需要概率建模和不确定性量化的研究
- 批次效应严重的多批次数据整合
- 多组学数据整合（totalVI, MultiVI）
- 与scvi-tools生态集成的分析流程

---

## 路线4: 图神经网络(GNN)

### 代表方法
GEARS, Celcomen, Cell Oracle

### 核心思想

GNN利用基因间的已知关系（如蛋白质相互作用网络、基因调控网络）构建图结构，通过消息传递机制在图上传播信息。每个基因是一个节点，基因间的关系是边，GNN学习节点表示和图级别的预测。

这种方法显式地利用生物学先验知识，将基因关系编码到模型架构中，使预测结果更符合生物学机制。

### 优势

- **先验知识整合**: 显式利用基因关系网络等生物学知识
- **可解释性强**: 注意力权重反映基因间的重要性关系
- **机制驱动**: 预测结果具有生物学机制解释
- **组合效应建模**: 擅长捕捉基因间的非加性相互作用
- **样本效率高**: 利用图结构可减少对大量训练数据的需求

### 局限

- **依赖先验网络**: 需要高质量的基因关系数据
- **网络不完整**: 现有基因网络可能不完整或有偏见
- **跨网络泛化**: 对未见过的基因或网络结构泛化能力有限
- **计算复杂度**: 大规模图的计算和存储开销较大

### 适用场景

- 需要机制解释的药物靶点发现
- 基因调控网络推断
- 组合扰动效应预测
- 空间转录组分析（Celcomen）

---

## 路线5: 语言模型/多模态

### 代表方法
CellHermes, scLAMBDA, GenePT, LPM

### 核心思想

这类方法利用大型语言模型（LLM）的语义理解能力，将单细胞数据转化为自然语言格式进行处理。例如，将高表达的基因列表转化为文本描述，然后使用预训练语言模型进行处理。

多模态方法进一步整合不同类型的生物学数据（如文本描述、蛋白质序列、相互作用网络），通过统一的语言表示进行融合。

### 优势

- **语义理解**: 利用LLM的隐含生物学知识
- **多模态融合**: 自然支持不同类型数据的整合
- **可解释性**: 自然语言接口提供更好的可解释性
- **零样本能力**: 部分方法无需训练即可使用
- **参数高效**: 可采用LoRA等参数高效微调技术

### 局限

- **信息损失**: 将数值数据转化为文本可能损失信息
- **依赖LLM质量**: 性能受限于基础LLM的能力
- **计算资源**: 大规模LLM的推理成本较高
- **领域适配**: 需要针对生物学领域进行专门优化

### 适用场景

- 需要整合异构数据类型的研究
- 自然语言交互的生物医学应用
- 利用文本知识库（如文献、数据库描述）的场景
- 快速原型开发和概念验证

## 路线选择建议

### 根据研究目标选择

| 研究目标 | 推荐路线 | 理由 |
|---------|---------|------|
| 高质量虚拟细胞生成 | 流匹配/扩散 | 生成质量最高，支持连续插值 |
| 大规模细胞图谱构建 | Transformer | 预训练模型可迁移到多种任务 |
| 批次效应校正 | VAE | scVI系列内置批次校正能力 |
| 药物靶点发现 | GNN | 可解释性强，整合先验知识 |
| 多模态数据整合 | 语言模型 | 自然支持异构数据融合 |

### 根据资源约束选择

| 资源情况 | 推荐路线 | 理由 |
|---------|---------|------|
| 计算资源充足 | Transformer/流匹配 | 可训练大规模模型 |
| 计算资源有限 | VAE/GNN | 计算成本较低 |
| 训练数据充足 | Transformer | 可充分利用大规模数据 |
| 训练数据有限 | GNN | 利用先验知识减少数据需求 |
| 需要快速推理 | VAE/Transformer | 单次前向传播 |
